### 一、构造函数与原型对象

#### 1. 构造函数的基本使用
```javascript
function CreateStudent(name, age) {
  this.name = name;
  this.age = age;
}

var studentA = new CreateStudent('yan', '18');
```

#### 2. new 关键字执行过程（4步）
| 步骤 | 操作                                           |
| ---- | ---------------------------------------------- |
| 1    | 创建一个新的空对象 `{}`                        |
| 2    | 将构造函数的 `this` 指向这个新对象             |
| 3    | 执行构造函数代码，给新对象添加属性             |
| 4    | 返回这个新对象（如果没有显式 return 其他对象） |

#### 3. 原型链三要素
```javascript
CreateStudent.prototype           // 构造函数的原型对象
studentA.__proto__                // 实例对象的隐式原型，指向构造函数的原型对象
CreateStudent.prototype.constructor  // 指向构造函数本身
```

**关系图：**
```
构造函数 CreateStudent
    ↓ (prototype属性)
原型对象 CreateStudent.prototype ← 实例对象 studentA 的 __proto__
    ↑ (constructor属性)
    └── 指向 CreateStudent
```

### 二、改变 this 指向的三种方法

#### 1. call、apply、bind 对比

| 方法    | 执行时机   | 传参方式                      | 返回值             | 使用场景           |
| ------- | ---------- | ----------------------------- | ------------------ | ------------------ |
| `call`  | 立即执行   | `fn.call(obj, arg1, arg2)`    | 函数执行结果       | 参数明确，个数固定 |
| `apply` | 立即执行   | `fn.apply(obj, [arg1, arg2])` | 函数执行结果       | 参数是数组形式     |
| `bind`  | 返回新函数 | `fn.bind(obj, arg1, arg2)`    | 绑定 this 的新函数 | 需要延迟执行       |

#### 2. 实际应用示例

**求数组最大值：**
```javascript
var arr = [1, 3, 4, 5, 6, 656, 7];
var max = Math.max.apply(Math, arr);
console.log(max); // 656
```

**bind 返回值：**
```javascript
const result = person.say.bind(person1)();
// 返回 person.say 函数执行后的返回值
// 如果没有显式 return，则返回 undefined
```

### 三、jQuery 插件开发模式

#### 1. 完整插件代码
```javascript
// 使用 IIFE 创建独立作用域
(function () {
  // 1. 构造函数 - 存储 DOM 元素
  function jQuery(selector) {
    this.selector = document.querySelector(selector);
  }

  // 2. 原型对象 - 存放公共方法
  jQuery.prototype = {
    click: function (fun) {
      this.selector.onclick = function () {
        fun();
      };
    }
  };

  // 3. 暴露工厂函数到全局
  window.myjQuery = function (selector) {
    return new jQuery(selector);
  };
})();
```

#### 2. 使用方式
```javascript
myjQuery('.btn').click(function () {
  console.log('点击了按钮');
});
```

### 四、IIFE（立即执行函数表达式）详解

#### 语法
```javascript
(function () {
  // 代码块
})();
```

#### 作用
| 作用             | 说明                 |
| ---------------- | -------------------- |
| **作用域隔离**   | 内部变量不会污染全局 |
| **模块化封装**   | 隐藏内部实现细节     |
| **避免命名冲突** | 多个插件可以共存     |
| **闭包保存状态** | 内部变量常驻内存     |

#### 为什么插件要用 IIFE？
- 防止全局变量污染
- 保护内部实现不被外部修改
- 只暴露必要的 API（`window.myjQuery`）

### 六、知识点总结表

| 分类          | 核心概念                                | 关键要点                                    |
| ------------- | --------------------------------------- | ------------------------------------------- |
| **构造函数**  | `new` 操作符                            | 4步流程：创建对象 → 绑定 this → 执行 → 返回 |
| **原型对象**  | `prototype`、`__proto__`、`constructor` | 三者相互关联，实现属性和方法共享            |
| **this 指向** | `call` / `apply` / `bind`               | call/apply 立即执行，bind 返回新函数        |
| **闭包**      | IIFE                                    | 隔离作用域，避免全局污染                    |
| **插件模式**  | 构造函数 + 原型 + 工厂函数              | 封装 DOM 操作，暴露统一 API                 |
| **链式调用**  | `return this`                           | 方法返回实例本身，支持连续调用              |

### 七、改进建议（支持链式调用和多元素）

```javascript
(function () {
  // 构造函数 - 支持多个元素
  function jQuery(selector) {
    this.elements = document.querySelectorAll(selector);
  }

  // 原型对象
  jQuery.prototype = {
    constructor: jQuery,  // 手动指定构造函数
    
    click: function (fun) {
      this.elements.forEach(element => {
        element.onclick = fun;
      });
      return this;  // 支持链式调用
    },
    
    css: function (prop, value) {
      this.elements.forEach(element => {
        element.style[prop] = value;
      });
      return this;
    }
  };

  window.myjQuery = function (selector) {
    return new jQuery(selector);
  };
})();

// 链式调用示例
myjQuery('.btn')
  .click(function () {
    console.log('点击了');
  })
  .css('color', 'red');
```

### 八、核心要点回顾

1. **构造函数 + new**：创建对象的模板
2. **原型对象**：共享方法，节省内存
3. **call/apply/bind**：动态改变 this 指向
4. **IIFE**：隔离作用域，封装插件
5. **工厂函数**：暴露简洁 API
6. **链式调用**：`return this` 实现

这套模式是 jQuery 和众多 JavaScript 库的经典实现方式！
